#JavaScript高级程序设计

##变量、作用域和内存的问题

###基本类型和引用类型
JavaScript不允许直接访问内存中的位置

  • 基本类型： Undefined、Null、Boolean、 Number、String (按值访问)
  • 引用类型：Object，Function，Date，Reg… (保存在内存中，操作引用)

函数中的参数都是按值传递
、、、
function setNames(obj) {
obj.name = “Baoman”;
obj = new Object(); //重写obj，变量应用的就是一个局部变量，而这个局部变量在函数执行完毕后立即被执行销毁
obj.name = “wu”
}
var person = new Object();
setName(person);
alert(person.name) //“Baoman”

类型检测typeof，instanceof

  • typeof用于检测基本类型，对于应用类型无能为力
    、、、
    var s = “abc”;
    alert(typeof s); //string
    、、、
  • instanceof用于检测引用类型
    、、、
    var color = [];
    alert(color instanceof Array); //true
    、、、
    ###执行环境及作用域
  1. 每个函数都有自己的执行环境 ——> 函数体有独立的作用域
  2. 执行环境类型总共有两类： 全局window对象和局部（函数）
  3. 没有块级作用域，使用for语句时尤其注意，但最新的es6已经解决了这个问题，使用let变量替换var如下代码：
