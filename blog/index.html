react bug 记录

发表于 2017-12-02 | 分类于 Bug记录

[日期]
2017-12-02

[项目]
华穗后台管理系统

[问题]
通过modalProps中的modalVisible属性控制modal的显示和隐藏，modalProps改变后，modal中显示的数据没有被改变
<JobModal {...modalProps} />
[原因]
react在脏数据检查时，如果dom结构没有改变，不会重新渲染组件。因此

[怎么发现的]
任意模块点击编辑和新增modal显示的内容一样。正确结果应该是，点击新增时，modal显示空的数据内容，点击编辑时应该从列表中将数据带入modal。但是，现在的结果是新增和编辑显示的内容一样。

[修复]
通过modalProps中的modalVisible属性控制modal的是否加载dom，react将重新渲染
{modalVisible && <JobModal {...modalProps} />}

第四章 变量、作用域和内存的问题

基本类型和引用类型

JavaScript不允许直接访问内存中的位置

  • 基本类型： Undefined、Null、Boolean、 Number、String (按值访问，存放在内存中的栈中)
  • 引用类型：Object，Function，Date，Reg… (保存在内存的堆中，操作引用)

函数中的参数都是按值传递

类型检测typeof，instanceof

  • typeof用于检测基本类型，对于应用类型无能为力

  • instanceof用于检测引用类型

执行环境及作用域

  1. 每个函数都有自己的执行环境 ——> 函数体有独立的作用域
  2. 执行环境类型总共有两类： 全局window对象和局部（函数）
  3. 没有块级作用域，使用for语句时尤其注意，但最新的es6已经解决了这个问题，使用let变量替换var如下代码：

垃圾收集

  • JavaScript具有自动垃圾收集机制，开发人员无需考虑垃圾回收问题
  • JavaScript有两种垃圾回收机制： 标记清除，引用计数
